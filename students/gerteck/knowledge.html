Tool/Technology

List the aspects you learned, and the resources you used to learn them, and a brief summary of each resource.

Developer Tools & Workflow

  • Git Worktrees: Learned to manage multiple checked-out branches simultaneously using git worktree.
    • Allows working on a long-running feature while quickly switching to a bug fix in a separate directory without stash/checkout overhead.
    • Facilitates running multiple versions of the app side-by-side for comparison and testing.
  • GitFlow branching strategy: A Git branching model designed around the project release cycle.
    • Uses main for production-ready code and develop as the primary integration branch for features.
    • Utilizes supporting branches like feature/ for new development, release/ for staging, and hotfix/ for immediate production repairs.
    • Used online resource to learn and also implemented it practically.

AI-Assisted Development & Architecture

  • AI Architecture Builder: Explored the use of AI to generate and iterate on system architecture diagrams and design patterns
    • Helps in visualizing complex component interactions and validating architectural logic before diving into code.

Claude Code & AI Agents

  • Claude Code (CLI), AntiGravity (Gemini), explored the use of AI to automate tasks and use AI Agents autonomous entities capable of executing multi-step developer workflows like codebase searching, editing, and testing.
    • Used to conduct labor-intensive migration processes from js to ts.
    • Used to come up with proof-of-concepts (e.g. integration of specific library) and prototyping.
  • Skills: I learned about Skills (Agentic Capabilities), which are specialized functions that extend an AI agent's utility.

Copilot Code Review

  • GitHub Copilot for Code Review: investigated and used it to assist in my role as the team lead. I focused on using AI to generate PR summaries and identify logic flaws or style inconsistencies, before conducting manual PR reviews.

Soft Skills and Knowledge

  • Team Leadership & Meeting Facilitation: Managed a developer team by leading weekly syncs, setting project agendas, and balancing roadmap planning with active technical problem-solving.
  • Code Review & Mentorship: Became more versed with the code review process, playing a stronger code peer reviewer role. Focused on identifying logic flaws, maintaining style consistency, and using feedback as a tool for team mentorship.
  • People Management: Focused on unblocking developers, resolving technical disagreements, and managing team bandwidth during release cycles.
  • Presentation Skills (PUMA Framework): Adopted the PUMA framework to deliver more impactful and structured technical presentations.
  • Slide Preparation & Preparation Tips: Learnt many different presentation techniques and tips to improve my presentation skills.
