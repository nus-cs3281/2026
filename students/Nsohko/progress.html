Week 7 UAT for migrated features Conducted UAT for key migrated features, including Feedback Sessions, Deadlines, and other related components, to verify correctness and ensure the migration changes worked as expected in practice. -
Week 8 Migrate GetDeadlineExtensionTest Migrated getDeadlineExtensionActionTest to use sqlentity.DeadlineExtension instead of DeadlineExtensionAttributes, continuing the shift away from datastore-era entity usage in tests. PR #13601
Week 9 Upgrade Liquibase Upgraded the Liquibase Gradle plugin to restore compatibility with Gradle 9 after Liquibase tasks such as liquibaseSnapshot and liquibaseDiffChangeLog started failing immediately at runtime. The root cause was the project using liquibase-gradle-plugin 2.2.1, which still depended on the removed Project.exec() API. The fix upgraded the plugin to 3.1.0, added the required liquibase-core buildscript dependency, and updated the Liquibase configuration syntax to the new 3.x format. This unblocked local database schema snapshot and diff generation using the standard Gradle workflow. PR #13677
Week 9 Update FeedbackResponseComment IDs to UUID Updated FeedbackResponseComment IDs across the codebase from generated integer values to UUIDs for consistency with the rest of the entities. Also added a defensive rollback in the SQL integration test base setup so if a previous test fails catastrophically and leaves a transaction open, subsequent tests can still run and be debugged properly. PR #13671
Week 10 Remove GAE dependence from cron jobs Removed the remaining GAE-specific authentication dependency from machine-triggered endpoints by adding bearer token validation for cron and worker endpoints. This made the endpoints compatible with external cron and task queue services instead of relying on App Engine specific headers. PR #13658
Week 11 Extract PR request checker script Moved the PR request checker logic out of the inline GitHub Actions workflow script into a separate JavaScript file to improve maintainability and make future checks easier to extend. Also updated the workflow to reference the extracted script and modernized the related GitHub Script action version. PR #13686
Week 11 Add explicit backend environment configuration Added support for explicitly setting the backend environment through configuration instead of inferring it from GAE-specific environment variables. This helps decouple the deployment logic from App Engine and makes production versus development setup more robust and predictable. PR #13668
Week 12 Preserve hyperlinks in plaintext email Fixed the plaintext email generation flow so hyperlinks are preserved instead of being stripped out during HTML-to-text conversion. This improves the usability of plain text email content, especially for messages that depend on session or recovery links. PR #13676
Week 12 Fix EmailGeneratorTestIT date handling Fixed EmailGeneratorTestIT after a hardcoded session date caused time-sensitive failures as the current date changed. The work also included auditing related tests and JSON fixtures to ensure date values are handled relative to the current time where appropriate, improving long-term test stability. PR #13701
Week 12 Identify account model decoupling issue Identified a broader design issue where accounts and related APIs still treat googleId as the stable login key, making multi-provider identity support awkward. Proposed a higher-level solution centered around introducing provider-based account identities and shifting account uniqueness toward provider-independent identifiers. -
Week 13 Migrate storage and logic layer away from Google ID Began migrating the storage and logic layers away from googleId as the core account identifier by introducing a more flexible identity model that supports provider-based authentication. This lays the groundwork for future multi-provider login support while reducing schema and API coupling to Google-specific concepts. PR #13755
